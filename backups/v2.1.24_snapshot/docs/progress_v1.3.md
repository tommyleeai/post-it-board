# 開發進度紀錄 v1.3 (Post It 數位白板)

**紀錄時間**: 2026-04-12  
**核心焦點**: UX 防呆機制、體驗升級與歷史紀錄系統 (Archive Manager)。

---

## 🚀 已完成功能與改動

### 1. 物理視覺感升級 (Physical Realism)
*   **尺寸與版面**：貼紙預設為 320x280 (約長寬兩倍)，改用固定寬度與 `max-height` 搭配內部滾動條，確保使用者輸入過多文字時貼紙不變形。
*   **膠帶隨機化生成 (Random Tapes)**：捨棄靜態的 `::before`，改由 JavaScript 每次新建卡片時動態生成 `<div class="note-tape">`，亂數控制厚度範圍 (0.8~1.0 透明度)、位置、旋轉角度，加上 20% 機率有髒污，15% 機率產生翹角效應，更具自然張貼感。

### 2. 直覺且容錯的 UI/UX 操作 (Failsafe UX)
*   **防呆快捷刪除**：將卡片的垃圾桶按鈕改為「滑鼠懸停 1 秒後浮出」。不需要跳出惱人的 confirm alert 就能刪除，同時大幅杜絕了誤點的可能。
*   **「蓋章完成」特效**：在單張設定面板中新增蓋章功能。點擊後會彈出一顆帶有不均勻印泥效果、甚至伴隨「重擊反彈」放大縮小動畫的紅色完成印章。蓋上後貼紙自動轉為無彩色的透明灰階，宣告任務死淨。
*   **4 秒延遲後悔機制 (Toast Undo)**：蓋上完成章後，通知列會有 4 秒的倒數，給予一個「復原」按鈕，幫助腦波弱或是按錯的使用者挽回剛剛的工作，點擊一鍵歸位。

### 3. 架構大翻新：歷史歸檔所 (Archive Manager)
*   **全新資料集合**：在 Firestore (`users/{uid}/postit_archived`) 新闢了歸檔儲存區。蓋章後的資料不再直接刪除，而是轉移過去並帶有一把 `completedAt` 的完工時間鑰匙。
*   **Modal 迷你清單牆**：在頂部控制列右方加了一個「歷史歸檔箱」按鈕 (`#btn-view-archive`)。為了美觀與快速辨識，把過往白板的卡片全部做成了迷你磁磚縮圖，顏色跟字型完全原汁原味繼承自歸檔前的一刻，並帶有過多文字底層漸層遮罩。
*   **永久性處置與撈回**：在歸檔區可對任一張卡片進行：
    *   `復原`：移回資料庫原位置，再次於白板出現。
    *   `永久刪除`：跳出最終 Confirm 彈窗，點擊後灰飛煙滅釋出資源。

### 4. 主題樣式層級機制 (Hierarchical Styling)
*   成功實作：`單張卡片手動設定 (Highest)` > `使用者帳號預設 (Overrides)` > `系統預定字體大小與色板 (Lowest)` 的層疊合併系統，在新建與編輯讀取階段順利串通。

## 🛠 未來潛在待辦或注意事項
*   **手機端相容 (Mobile Response)**：目前的 `hover` 刪除機制可能對純觸控設備不夠友善，之後若推向手機，需針對 touch UI 微調 (或單純依靠內頁的 `刪除` 按紐)。
*   **時效性留言提醒**：目前尚未切入「指定時間不斷抖動發出聲音提醒」的即時排程檢測邏輯，預計規畫在下一階段中實作背景定時器 (`setInterval` 掃描或 Service Worker 推播)。
